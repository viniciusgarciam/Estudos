# 📚 Junior Dev Study Plan – Abril a Julho 2025

## 🎯 Objetivo: Estar pronto para atuar como Dev Júnior até Julho de 2025

### ⏳ Tempo disponível
- Segunda a sexta: 4h/dia
- Finais de semana e feriados: 8h/dia
- Total estimado: ~384 horas até julho

---

## 🧭 Diretrizes do Mentor (integradas ao plano)

- **Comunicação e alinhamento**: mais importante que técnica
- **Gestão de tempo**: essencial para home office
- **Fundamentos de computação/internet/cloud**: base sólida antes de GCP/AWS
- **Desenvolvimento com Go e Python**
- **Bancos de dados SQL e NoSQL**
- **Inglês técnico**: commits, READMEs, termos e leitura

---

## 📆 Fases do Estudo

### 📌 Fase 1: Abril 15–30 – Fundamentos e Organização
**Objetivo**: Revisar fundamentos + começar API com Go

#### Tópicos:
- [x] Revisar Go (tipos, structs, funções, erros)
- [x] Reforçar Git + GitHub (commits, branch, README em inglês)
- [x] Início da API com Gin (roteamento, handlers simples)
- [x] SQL básico (CRUD, SELECT/INSERT/UPDATE/DELETE)
- [x] Projeto: `task-manager-api` (começar)
- [x] Leitura diária: 15–20min (inglês e livros de comunicação)

---

### 📌 Fase 2: Maio – Projeto completo + Banco de Dados
**Objetivo**: Criar API real com banco, autenticação e deploy local

#### Tópicos:
- [ ] Gin + GORM + PostgreSQL (CRUD completo)
- [ ] JWT Authentication (login, refresh token)
- [ ] Organização por camadas (routes, handlers, services)
- [ ] Docker: app + banco
- [ ] Deploy local (Docker Compose ou Render)
- [ ] README completo em inglês
- [ ] Escrever issues e commits em inglês

---

### 📌 Fase 3: Junho – Cloud, Testes e Consolidação
**Objetivo**: Deploy na nuvem + testes + projeto 2

#### Tópicos:
- [ ] Deploy com GCP (Cloud Run)
- [ ] Configurar Dockerfile e docker-compose.yaml
- [ ] Cloud: variáveis de ambiente, storage, logging
- [ ] Testes com `testing` do Go (unitários simples)
- [ ] NoSQL com MongoDB (básico)
- [ ] Projeto 2: `simple-agenda-api` ou `blog-api`
- [ ] Leitura de documentação oficial (Go, Gin, GCP)

---

### 📌 Fase 4: Julho – Revisão final e Portfólio
**Objetivo**: Revisar tudo, refatorar projetos e preparar para aplicar

#### Tópicos:
- [ ] Revisar pontos fracos (Go, SQL, deploy)
- [ ] Refatorar projetos principais
- [ ] Revisar testes, autenticação e organização do código
- [ ] Melhorar documentação e READMEs
- [ ] Criar um portfólio com links, prints, e explicações
- [ ] Simular entrevista técnica (explicar seu código)
- [ ] Checklist: estou pronto?

---

## 📌 Estrutura diária sugerida

### Segunda a sexta (4h/dia)
- 1h: Leitura (livros, artigos técnicos, inglês)
- 2h: Codificação prática
- 1h: Anotações, revisão, commits/documentação

### Fins de semana e feriados (8h/dia)
- 1h: Leitura
- 5h: Codificação + evolução dos projetos
- 2h: Revisões profundas (cloud, testes, debug)

---

## ✅ Resultados esperados em Julho

- 2 projetos completos com README em inglês
- GitHub limpo e profissional (commits, branches, issues)
- Domínio de Go, REST APIs, SQL, deploy básico em cloud
- Comunicação técnica clara (oral e escrita)
- Pronto para aplicar em vagas de Dev Júnior

> 🎯 Foco em clareza, consistência e entrega. Você não precisa saber tudo — precisa saber construir, explicar, colaborar e aprender rápido.
